# 第5回 コマ1 演習：単体テスト入門（Java / JUnit 5）

自販機を題材に、単体テストを「書く」演習の配布物です。
**コマ1ではGitは使いません。** zip をダウンロードして展開し、VSCode で開くだけです。

## 準備（最初の1回だけ）

1. この zip を **ダウンロード → 展開（解凍）** する
2. VSCode で「**フォルダーを開く**」→ 展開した `test-lesson-01-unittest` を選ぶ
3. 左の **エクスプローラー** にファイルが並べばOK（「このフォルダーを信頼しますか？」は「信頼」）
4. ターミナル（`Ctrl + @`）で次を実行：

   ```
   mvn test
   ```

   ※ 初回は JUnit の取り寄せに少し時間がかかります。

## ファイルの場所

```
src/main/java/vending/      ← テストされる側（読むだけ・完成済み）
    Calc.java                 足し算の部品（写経の対象）
    Drink.java                飲み物（名前を持つ）
    VendingMachine.java       自販機（水150 / お茶120 / コーヒー130=売切れ）

src/test/java/vending/      ← あなたが手を入れるテスト
    CalcTest.java             演習①-2：写経（手で打つ）
    VendingMachineTest.java   演習①-3：穴埋め（_____ を仕様書の値で埋める）
```

## 進め方

**演習①-2（写経）** … `CalcTest.java` を開き、コメントの手順どおり自分で打つ。
保存して `mvn test`、Green になればOK。

**演習①-3（穴埋め）** … `VendingMachineTest.java` の `_____` を、テストケース仕様書の値で
1つずつ埋める。ケース1は完成例。ケース2・5・6を埋めて保存し、`mvn test` で **4本 Green** を目指す。

## つまずいたら（Red / エラーになったら）

| 症状 | 原因・対処 |
|------|------------|
| `_____ cannot be resolved` | 穴（`_____`）がまだ埋まっていない。保存（Ctrl+S）も忘れずに |
| `expected: <50> but was: <80>` | 仕様書の値と違う。入れた金額・おつりを見直す |
| `mvn: command not found` | Maven が未インストール。VSCode の Java 拡張からテスト実行でもOK |
| 文字化け | ファイルを UTF-8 で開き直す |
