# 第5回 コマ1 演習：単体テスト入門（Java / JUnit 5）

自販機を題材に、単体テストを「動かす → 書く」演習です。

---

## 事前準備（最初の1回だけ）

1. **親フォルダを作る**
   PC上に、授業用の親フォルダを1つ作成する。
   （3コマ分の演習は、すべてこの親フォルダの中で行います）

2. **VSCode で開く**
   VSCode の「フォルダーを開く」→ いま作った親フォルダを選ぶ。

3. **GitHub でリポジトリを作成**
   GitHub で、自分用のリポジトリを1つ作る。

4. **クローンする**
   VSCode のターミナル（`Ctrl + @`）で、次を実行する：

   ```
   git clone https://github.com/ユーザー名/リポジトリ名.git
   ```

   → 親フォルダの中に演習用フォルダができる。それを VSCode で開く。

> ※ 初回のテスト実行は、JUnit の取り寄せに少し時間がかかります。

---

## ファイルの場所

```
src/main/java/vending/      ← テストされる側（読むだけ・完成済み）
    Calc.java                 足し算の部品
    Drink.java                飲み物（名前を持つ）
    VendingMachine.java       自販機（水150 / お茶120 / コーヒー130=売切れ）

src/test/java/vending/      ← テストを置くところ
    CalcTest.java             演習①-2：実行して確認（すでに書いてある）
    VendingMachineTest.java   演習①-3：穴埋め（_____ を埋める）
```

---

## 演習①-2：用意されたテストを動かす

1. `CalcTest.java` を開く
   → `add(2, 3)` のテストが、**すでに書いてあります**。
2. ターミナルで `mvn test` を実行する。
3. **Green（成功）** になればOK。

ここでは「テストが通る ＝ プログラムが期待どおり動いている」ということを、
まず自分の目で確認します。

---

## 演習①-3：自販機テストの穴埋め

`VendingMachineTest.java` の `_____` を、テストケース仕様書の値で1つずつ埋めます。

- ケース1は **完成例**（書き方の見本）。
- ケース2・5・6 を埋めて保存（`Ctrl + S`）。
- ターミナルで `mvn test` → **4本すべて Green** を目指す。

---

## つまずいたら（Red / エラーのとき）

| 症状 | 原因・対処 |
|------|------------|
| `_____ cannot be resolved` | 穴（`_____`）がまだ埋まっていない。埋めて保存する |
| `expected: <50> but was: <80>` | 仕様書の値と違う。入れた金額・おつりを見直す |
| `mvn: command not found` | Maven が未インストール。VSCode の Java 拡張のテスト実行でもOK |
| 文字化け | ファイルを UTF-8 で開き直す |
